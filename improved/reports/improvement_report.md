# 데이터 분석 경진대회 사후 개선 실험 결과 보고서

---

## 1. 문제 정의 / 출발점

본 보고서는 한국데이터정보과학회 & SAS KOREA 제2회 데이터 분석 경진대회 종료 이후 진행한 사후 개선 실험의 기록이다.

대회 종료 시점 성적은 Churn AUC 0.7943 / Total Score 0.46683이었다. 5주간 EDA·피처 엔지니어링·앙상블·하이퍼파라미터 튜닝을 모두 시도해 보았음에도 1주차 베이스라인(0.788)에서 0.794까지 약 0.006 개선에 그쳤고, 2주차 이후로는 사실상 수렴했다. 당시 막연히 느꼈던 세 가지 미진함이 이 실험의 출발 가설이 되었다.

1. **피처 과설계**(50개+)
2. **변수 간 상관관계를 점검하지 않고 방치**
3. CV 점수와 리더보드 점수의 괴리를 추적하지 않은 **검증 설계 부실**

이에 따라 모델 성능 개선 실험을 설계하기 이전에, 위 가설의 진위여부를 먼저 규명하기로 했다.

---

## 2. 진단

위 항목에 정의한 가설을 각각 **증상 → 의심 원인 → 확인 방법**으로 분해하고 데이터로 검증했다.
개선 과정 중 가장 중요한 단계이며, 결과적으로 대회 진행 당시 의사결정의 상당수가 **틀린 전제 위에 있었음**이 드러났다.

- **진단 (1) 피처 과다 → 다중공선성으로 importance 왜곡 → 상관행렬·null importance 점검**

split-based feature importance는 상관 피처가 있을 때 한쪽으로 쏠려 의사결정을 오염시킨다.

타겟을 무작위로 섞은 null importance와 실제 importance를 z-score로 비교한 결과, 기존에 핵심 피처로 여겼던 `fin_asset_trend_score`가 **Churn·LTV 양쪽에서 noise(z≈-0.9)**로 판정됐다.

단변량 분리력은 1위였으나 다른 금융 피처와 정보가 겹쳐 모델 기여가 0이었다. (전형적인 split-importance 착시)

- **진단 (2) CV-LB 괴리 → Train/Test 분포 불일치 의심 → adversarial validation**

Train=0/Test=1 라벨로 분류기를 학습해 두 분포의 구분 가능성을 측정한 결과 **AUC 0.4995**였다. 동전 던지기 수준으로 구분 불가, 즉 **분포 불일치는 존재하지 않았음**을 시사한다.

특히 "CV-LB 괴리의 주범"으로 지목됐던 `is_married` 변수를 실제로 집계해 보니 **Train 65.1% / Test 64.7%로 거의 동일**했다. 주차 기록에 적힌 "Train 95% vs Test 65%"는 **집계 오류**였고, "제거 시 AUC -0.007"이라는 포함 근거도 noise였다.

- **진단 (3) 거래 피처 신호 확인 → Mann-Whitney U + effect size**

이탈/유지 그룹 간 거래 피처 31종을 검정한 결과, 유의미한 분리력을 가진 피처는 거의 없었고 신호는 금융 자산 프로파일(`card_loan_amt`, `total_deposit_balance`, `card_cash_service_amt`)에 집중돼 있었다.

- **진단 (4) 평가 지표 재해석**

안내문 공식 `Score = 0.5×AUC + 0.5×1/(1+log(SE(LTV)))`를 상위 6팀 실측치로 역산한 결과, 로그는 **상용로그(log10)**, RMSE는 **원본(원) 스케일**임이 확인됐다. (자연로그/log스케일로는 실측 Score가 재현되지 않음)

134만 원 스케일에서 LTV term은 log10 함수의 평탄 구간에 있어, **RMSE를 5만 원 줄이는 것보다 AUC를 0.001 올리는 것이 약 6배 더 가치 있다.** 대회 당시 LTV 2-stage 모델링에 할애한 시간은 최종 Score에 거의 기여하지 않았다.

## 3. 모델 개선 실험 설계

위 진단 결과를 바탕으로 모델 개선 실험을 진행한다. 여러 변수를 동시에 바꾸면 무엇이 효과였는지 분리할 수 없으므로, 한 번에 하나씩만 변경하는 통제된 실험으로 설계했다.

1. **검증 스킴 정비**

모든 실험을 5-fold StratifiedKFold로 고정하고, adversarial validation으로 CV 신뢰성을 먼저 확보했다. 이 과정이 생략되면 이후 단계가 무의미하다는 것이 본 개선 실험에서 얻은 가장 큰 깨달음이다.

2. **데이터 검증 기반 피처 선택**

null importance로 "진짜 신호(z>3) / 애매한 신호 / noise"를 통계적으로 분류하고, full(전체) / denoised(검증 통과한 핵심 10개) / core(신호 3개) 세 가지 피처셋을 **실측 비교**했다.

3. **모델 다양성 도입**

앙상블 다양성을 확보하기 위해, 기존 LGB+XGB만 쓰던 구성에 **CatBoost·RandomForest를 추가**했다. (학습 메커니즘이 다른 모델은 예측 오차의 상관이 낮아 앙상블 이득이 커짐)

4. **타겟 변환**

LTV의 log1p 변환이 왜도를 -1.14로 과교정함을 확인하고, cbrt(세제곱근, 왜도 +0.14)와 비교했다.

## 4. 개선 결과 및 실험별 기여도 분해

대회 종료 당시 0.7943 → 사후 개선 결과 0.79824. **(AUC +0.0039)**
최종 Score는 0.46683 → 0.46904로 개선됐고, 시작 단계에서 설정한 목표(Score 0.469)를 달성했다.

단계별 단독 효과는 다음과 같다.

| 단계 | 변경 내용 | ΔAUC | 해석 |
|---|---|---|---|
| 출발점 | 대회 종료 시점 | — | 0.79430 |
| v1 | 4모델 다양성 앙상블 베이스라인 | **-0.00146** | 핵심 피처만 쓴 경량 버전 (일시 하락) |
| v2 | **Optuna 튜닝** + 다양성 강제주입 | **+0.00450** | **개선의 주동력 — 튜닝** |
| v3 | 다양성주입 제거 + RF튜닝 + 시드배깅 | **+0.00083** | XGB 회복(+0.0097)이 견인 |
| v4 | 거래피처 재설계 + 검증 내장 | **+0.00006** | 사실상 수렴 |

가장 큰 성능 개선 효과는 **Optuna 하이퍼파라미터 튜닝(+0.0045)**에서 나타났음을 확인할 수 있다.
LGB가 단독으로 +0.012 상승했는데, 신호가 소수인 데이터에서 기존의 복잡한 트리(num_leaves=31)가 노이즈를 외우다 과적합하던 것을 단순·보수적 트리(num_leaves=11, 강한 정규화)로 교정한 결과로 판단된다.
단, 대회 당시에도 Optuna 튜닝을 시도했으나 AUC가 오히려 악화(-0.0021)되어 기각한 기록이 있다. 따라서 "파라미터를 바꾸지 않은 것이 손해였다"는 단순한 해석으로 결론짓는 것에는 주의가 필요하다. 이 결과에서 제기할 수 있는 의문점은 "왜 같은 튜닝이 대회 때는 실패하고 사후 실험에서는 효과를 냈는가"이며, 이에 대한 확인은 이어지는 [5. 가설 검증] 단계에서 진행했다.

추가로, 효과가 없었거나 마이너스였던 시도는 아래와 같다.

- **피처 서브셋 분할(v2)**: 모델 간 상관을 낮추기 위해 각 모델에게 서로 다른 피처 부분집합(70~80%)을 할당하여 다양성 강제 주입을 시도했으나, OOF 상관은 0.95→0.99로 오히려 상승했고 XGB는 0.796→0.788로 망가졌다. 신호가 소수 피처에 집중된 데이터에서는 어떤 서브셋을 뽑아도 핵심 피처가 포함돼 다양성이 만들어지지 않았다. v3에서 이를 제거하자 XGB가 +0.0097 회복됐다.
- **거래 피처 재설계(v4)**: RFM·거래간격 변동성·카테고리 집중도(HHI)·채널전환·트렌드 비율 등 19개를 새로 만들어 null importance로 검증한 결과, **거래 기반 피처는 전부 noise(z<0)**였다. 살아남은 2개(cardloan_to_deposit z=20.5, card_debt_total z=7.2)는 거래가 아니라 금융 상호작용이었으나, 기존 피처와 중복되어 앙상블 기여는 없었다.
- **피처셋 비교**: full(43개) 0.7963 < denoised(10개) 0.7975. 전체 피처를 다 넣었을 때 가장 나빴고, 핵심 10개로 줄였을 때 가장 높은 수치를 기록했다. 이는 노이즈 피처가 모델의 일반화(Test) 성능 하락의 원인임을 증명하며, 가설(1)의 근거가 된다.
- **LTV cbrt 변환**: 원본스케일 RMSE를 149만→141만으로 줄였으나, 진단(4)에서 확인하였듯 최종 Score 기여는 미미했다.

## 5. 가설 검증 (AB 테스트)

위 결과에서 제기된 "Optuna 튜닝의 성능 개선 효과 차이"에 대한 의문점을 추측으로 두지 않고 통제된 실험으로 검증했다.

1. **검증 대상 가설**: "튜닝의 성능 개선 효과 차이의 원인은 **"피처 정리·검증 설계·신호 진단 작업 선행 여부"**이다."

2. **설계**: 두 피처셋에 동일한 튜닝을 적용해 효과를 비교하는 2×2 구조이다.
미정돈 피처셋(X)은 대회 종료 당시 피처 집계 코드를 그대로 실행한 5주차 실제 피처셋(피처 49개, is_married 오류·거래피처 다수·캡핑·상호작용 포함)이며, 정돈 피처셋(Y)은 진단을 통과한 10개 피처(denoised)이다. 정확한 비교를 위해 두 피처셋의 Optuna 탐색공간·trial 수(60)·시드·CV 분할을 모두 동일하게 적용했다.
이에 더해, 다음 네 가지 측정값을 각각 A·B·C·D로 표기한다.
- A = 미정돈 피처셋(X)을 튜닝 전 파라미터로 학습한 AUC
- B = A와 같은 피처셋을 튜닝한 AUC
- C = 정돈 피처셋(Y)을 튜닝 전 파라미터로 학습한 AUC
- D = C와 같은 피처셋을 튜닝한 AUC
따라서 (B−A)는 미정돈 피처셋에서의 튜닝 효과, (D−C)는 정돈 피처셋에서의 튜닝 효과이며, 두 값의 차이가 "피처셋 정돈이 튜닝 효과를 얼마나 키웠는가"를 나타낸다.

3. **결과**

| 대조군 | 튜닝 전 | 튜닝 후 | ΔAUC(튜닝 효과) |
|---|---|---|---|
| X = 5주차 피처 (49개) | 0.78623 | 0.79690 | **+0.01067** |
| Y = denoised (10개) | 0.78448 | 0.79751 | **+0.01304** |

- X(49개) 튜닝 결과: min_child_samples=22, colsample=0.89  (정규화 약함)
- Y(10개) 튜닝 결과: min_child_samples=46, colsample=0.51  (정규화 강함)

**(D−C) − (B−A) = +0.00237**라는 유의한 차이로, 정돈된 피처셋에서 튜닝 효과가 더 크게 나타나는 것을 확인하였다.

이 효과의 메커니즘은 각 대조군의 best_params 비교에서 드러난다. 미정돈 피처셋(X)에서 Optuna가 수렴한 min_child_samples는 22로, 정돈 피처셋(Y)의 0.5배 수준이다.
이는 49개 피처가 만드는 고차원 탐색 공간에서 60 trial로는 "강한 정규화 + 소수 핵심 피처 집중"이라는 최적 영역에 도달하지 못하고 애매한 지점에 머물렀음을 시사한다. 반면 Y는 탐색 공간이 작아 동일한 trial 예산으로 제대로 된 정규화 강도를 찾아냈다.

이를 통해 얻을 수 있는 핵심 시사점은 두 가지다.
**첫째, 튜닝 효과의 차이(+0.00237)는 튜닝 기법이 아니라 탐색 난이도에서 비롯되었다.** 알고리즘과 trial 예산이 동일해도, 노이즈가 탐색 공간을 부풀리면 튜닝은 최적해에 도달하지 못한다.
**둘째, 이는 trial 수를 늘리면 X에서도 Y에 근접할 가능성을 함의한다.** 즉 미정돈 피처셋의 튜닝 효과 부진은 원리적 한계가 아니라 탐색 예산의 문제일 수 있으며, 이 경우에도 "피처를 먼저 정리하는 것이 동일 예산으로 더 효율적"이라는 결론은 유지된다. 피처셋 정돈은 튜닝의 탐색 비용을 낮추는 선행 작업인 셈이다.

*한계: 본 재현 실험은 LGB 단독·시드 차이로 대회 절대값(0.7943)을 0.7862로 근사 재현함. 절대값은 완전히 일치하지 않으나, 동일 조건 내 상대 효과를 비교한 것이므로 결론의 방향성에는 영향이 없음.*

## 5. 한계 및 배운 점

1. **AUC가 0.798에서 멈춘 이유**

모델링 v2부터 v4까지 세 번 연속 0.798 부근에서 수렴했고, v4에서 만들 수 있는 모든 피처를 생성·검증했음에도 금융 3~4개 외에는 전부 noise였다. 즉 **이 피처 공간에서 0.798은 데이터 한계에 가깝다.** 경진대회 최종 결과 발표 당시, 상위 10위권의 AUC가 0.799대에 좁게 밀집된 양상을 보였다는 것이 본 결론의 근거이다.

0.799 이상의 성능 향상은 거래 피처 발굴 등과 같은 구조적 돌파가 아닌, 앙상블 다양성·시드 안정화·미세 튜닝의 누적을 비롯하여 상당 부분 분산(시드·fold에 따른 변동)의 영향으로 결정되는 것으로 판단된다. 더 짜내는 것은 새 신호 발굴이 아니라 리더보드 과적합에 가까워, 정석적 관점에서 여기서 멈추는 것이 옳다고 판단했다.

2. **추가로 시도할 수 있는 것**

Shallow Neural Network 등과 같이 학습 메커니즘이 더 이질적인 모델로 앙상블 상관을 0.99 아래로 낮추거나, 더 넓은 시드 배깅을 통해 분산을 안정화하는 실험을 추가로 고려할 수 있다. 기대 이득은 0.0005 미만으로 추정되나, "0.798 수렴은 데이터 한계"라는 결론의 근거 확보 목적으로 시도 가능하다.

3. **실험 결과의 시사점**

이번 실험을 통해 얻은 결론이자 가장 큰 배움은 **데이터 검증 및 피처 정돈이 선행되어야 튜닝의 효과가 증폭되는 순차적 관계**라는 것이다.

개선 실험 전 단계에 걸쳐 확인한 근본적인 문제점은 **모델 자체를 의심하기 이전에 데이터 검증(adversarial validation)과 신호 진단(null importance)을 선행하지 않았다는 것**이다. 대회 진행 당시 5주차에 Optuna 튜닝을 시도했음에도 불구하고 성능 개선 효과를 확인할 수 없었으나, 이는 튜닝 기법의 문제가 아닌 검증되지 않은 피처셋을 기반으로 작업한 결과였다. 검증 설계와 신호 진단을 모델 설계 이전에 선행했다면, is_married 오류를 1주차에 바로잡고, 거래 피처가 무신호임을 사전에 확인하고, 평가 지표에 AUC 비중이 훨씬 크다는 사실을 토대로 진행 방향성을 점검하여 보다 효율적이고 다양한 방식으로 실험을 진행할 수 있었을 것이라는 아쉬움이 남는다.

이에 따라 본 개선 실험 과정에서 진행한 데이터 검증 단계를 향후 유사한 예측 모델링 과제에도 표준 베이스라인으로서 적용할 계획이다. 분포 점검·신호 검증·피처 정돈 과정을 선행 단계로 고정함으로써, 문제 요소를 놓치지 않고 초기에 통제할 계획이다.
